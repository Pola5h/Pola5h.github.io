PHP Coding Standards Every Bangladesh Developer Should Follow

Published: July 15, 2024 by Kamruzzaman Polash

Following consistent coding standards is essential for Bangladesh developers working in teams or building maintainable applications. Here are the key standards every PHP developer should adopt.

PSR Standards

PHP-FIG (Framework Interop Group) standards ensure code compatibility across different PHP projects.

PSR-1: Basic Coding Standard

  • Use only opening tags <?php
  • Use UTF-8 without BOM
  • Class names in PascalCase
  • Method names in camelCase
// Good class UserService { public function getUserById($id) { return User::find($id); } } // Bad class user_service { public function get_user_by_id($id) { return User::find($id); } }

PSR-2: Coding Style Guide

  • Use 4 spaces for indentation (not tabs)
  • Line length should be 120 characters or less
  • Opening braces on same line for methods
  • One statement per line
// Good formatting if ($condition) { $result = processData($data); return $result; } // Bad formatting if($condition){ $result=processData($data);return $result; }

Laravel-Specific Standards

Naming Conventions

  • Controllers: PascalCase with "Controller" suffix
  • Models: Singular PascalCase
  • Variables: camelCase
  • Database tables: plural snake_case
// Controllers class UserController extends Controller {} // Models class User extends Model {} // Variables $userData = []; $isActive = true; // Database tables users, blog_posts, user_profiles

Method Documentation

Always document your methods, especially for team projects:

/** * Create a new user account * * @param array $data User registration data * @return User The created user instance * @throws ValidationException If validation fails */ public function createUser(array $data): User { // Implementation }

Tools for Code Quality

PHP CodeSniffer

Automatically check coding standards:

// Install composer require --dev squizlabs/php_codesniffer // Check code ./vendor/bin/phpcs --standard=PSR2 app/ // Auto-fix issues ./vendor/bin/phpcbf --standard=PSR2 app/

PHP-CS-Fixer

Automatically fix coding standard violations:

// Install composer require --dev friendsofphp/php-cs-fixer // Fix files ./vendor/bin/php-cs-fixer fix app/

Best Practices for Bangladesh Teams

1. Use English for Code

  • Variable names in English
  • Comments in English
  • Function names in English

2. Consistent Error Handling

// Good: Consistent exception handling try { $user = $this->userService->createUser($data); } catch (ValidationException $e) { return response()->json(['errors' => $e->errors()], 422); } catch (Exception $e) { Log::error('User creation failed: ' . $e->getMessage()); return response()->json(['error' => 'Internal error'], 500); }

3. Security Standards

  • Always validate user input
  • Use prepared statements for database queries
  • Sanitize output data
  • Follow Laravel security best practices

Team Collaboration

For Bangladesh development teams:

  • Set up EditorConfig for consistent formatting
  • Use Git hooks to enforce standards
  • Regular code reviews
  • Shared coding standards document

Following these coding standards will make you a more professional developer and improve collaboration in Bangladesh tech teams. Start with PSR standards and gradually adopt Laravel-specific conventions.
