1. @hasStack Blade Directive: Clean Template Organization

The most exciting feature in Laravel 12.39 is the new @hasStack Blade directive. This directive allows you to conditionally render HTML elements only when a Blade stack contains content, preventing empty wrapper elements and improving template cleanliness.

1.1. How @hasStack Works

Before Laravel 12.39, you might have empty HTML elements when stacks were unused. The @hasStack directive solves this elegantly:

@push('scripts')
<script>console.log('Custom script');</script>
@endpush

@hasstack('scripts')
<script src="https://cdn.example.com/app.js"></script>
@stack('scripts')
@endif

This ensures the script tag and CDN reference only render when the 'scripts' stack has content.

1.2. Real-World Use Cases

  • Conditional CSS/JS Loading: Only load assets when needed
  • Dynamic Meta Tags: Add meta tags only when content exists
  • Component Flexibility: Make components more adaptable
  • Performance Optimization: Reduce unnecessary HTML output

2. Route List Middleware Filtering: Advanced Debugging

Laravel 12.39 introduces a powerful new feature for the route:list command. You can now filter routes by middleware, making it easier to debug and understand your application's routing structure.

2.1. Basic Middleware Filtering

Use the new --middleware option to filter routes:

php artisan route:list --middleware=api
php artisan route:list --middleware=auth
php artisan route:list --middleware=ThrottleRequests

2.2. Advanced Filtering Options

  • Exact Match: --middleware=auth
  • Partial Match: --middleware=Throttle
  • Multiple Filters: Combine with other route:list options

2.3. Debugging Benefits

This feature is invaluable for:

  • API Development: Quickly see all API routes
  • Security Audits: Check authentication middleware usage
  • Performance Analysis: Identify rate-limited routes
  • Maintenance: Understand middleware dependencies
