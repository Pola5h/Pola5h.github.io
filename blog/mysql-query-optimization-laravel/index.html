MySQL Query Optimization in Laravel: Advanced Techniques

By Kamruzzaman Polash • November 15, 2024 • 12 min read

Database performance is critical for Laravel applications. Slow queries can kill user experience and server resources. In this guide, I'll share practical optimization techniques I've used in production applications.

The N+1 Query Problem

The most common performance killer in Laravel applications. Here's how it happens and how to fix it:

// BAD: This creates N+1 queries $users = User::all(); foreach ($users as $user) { echo $user->profile->bio; // Each iteration hits the database } // GOOD: Use eager loading $users = User::with('profile')->get(); foreach ($users as $user) { echo $user->profile->bio; // Profile already loaded }

Query Optimization Techniques

1. Use Select Specific Columns

// Instead of loading everything $users = User::all(); // Select only what you need $users = User::select('id', 'name', 'email')->get();

2. Implement Database Indexing

Add indexes to frequently queried columns:

// In migration Schema::table('users', function (Blueprint $table) { $table->index('email'); $table->index(['status', 'created_at']); });

3. Use Query Caching

// Cache expensive queries $popularPosts = Cache::remember('popular_posts', 3600, function () { return Post::where('views', '>', 1000) ->orderBy('views', 'desc') ->take(10) ->get(); });

Laravel Query Builder Tips

Chunk Large Datasets

// Process large datasets in chunks User::chunk(1000, function ($users) { foreach ($users as $user) { // Process each user } });

Use Lazy Collections

// For memory-efficient processing User::lazy()->each(function ($user) { // Process without loading all records into memory });

Monitoring Query Performance

Use Laravel Telescope or enable query logging to identify slow queries:

// Enable query logging DB::enableQueryLog(); // Your queries here // Check executed queries $queries = DB::getQueryLog(); dd($queries);
Pro Tip: Always test your queries with realistic data volumes. A query that works with 100 records might fail with 100,000.

Conclusion

Query optimization is an ongoing process. Start with solving N+1 queries, add proper indexing, and implement caching strategies. Monitor your application's performance and optimize bottlenecks as they appear.
